## 1. 游戏概述

### 1.1 一句话描述

把逃离塔科夫的"搜刮→战斗→撤离→治疗"核心循环做成回合制文字策略游戏。

### 1.2 游戏类型

回合制策略 Roguelike / 文字搜打撤

### 1.3 目标平台

浏览器（PC端优先，移动端可打开但体验一般）

### 1.4 目标用户

- 对"搜打撤"玩法感兴趣但没时间/没设备玩FPS的玩家
- 喜欢策略决策胜过操作反应的玩家
- 碎片时间玩家（一局5-10分钟）

### 1.5 差异化定位

我不是在做"塔科夫网页版"，那个做不出来也太复杂。我做的是**把塔科夫的核心决策体验剥离出来**：

| 塔科夫提供 | 我的版本 |
|-----------|---------|
| 搜刮的赌博感 | ✅ 稀有度随机 + 高价值区 + 上锁房间 |
| 撤离的紧张感 | ✅ 回合限制 + 超时=死亡 + 装备全丢 |
| 伤势的持续感 | ✅ 局外需手动治疗 + 断裂需手术包 |
| 装备管理的取舍 | ✅ 背包容量有限 + 保险箱有限 |
| 经济循环 | ✅ 商人+跳蚤+藏身处升级 |
| FPS枪法 | ❌ 不需要 |
| 地图记忆 | ❌ 地图每次随机生成 |
| 3D建模/光影/音效 | ❌ 只有emoji和CSS |

核心句：**"保留策略深度，砍掉执行门槛。"**

## 2. 核心循环

### 2.1 主循环图

```
┌──────────────────────────────────────┐
│              🏠 主界面               │
│                                      │
│  · 查看/治疗身体伤势（手动用药品）     │
│  · 吃食物补充体力/水分                │
│  · 购买装备（商人/跳蚤市场）           │
│  · 升级藏身处（消耗搜刮材料）          │
│  · 查看/接取任务                      │
│  · 管理保险箱                         │
│                                      │
└────────────┬─────────────────────────┘
             │ ① 选地图 → 付入场费 → 确认装备
             ▼
┌──────────────────────────────────────┐
│            🔫 局内（突袭）            │
│                                      │
│  回合制网格移动（6×6 ~ 12×12）        │
│                                      │
│  每回合可执行：                       │
│  ├ 移动到相邻格                       │
│  ├ 搜刮物资（踩上自动弹窗）            │
│  ├ 战斗（点击部位瞄准）                │
│  ├ 使用医疗品/吃食物                   │
│  ├ 丢弃物品/存入保险箱                 │
│  ├ 强行破门（30%概率，失败受伤）        │
│  └ 走到撤离点 → 结束                  │
│                                      │
│  移动消耗：2体力 + 1水分/步            │
│  体力归零 → 每步-5HP饥饿伤害           │
│                                      │
└────────────┬─────────────────────────┘
             │
      ┌──────┴──────┐
      ▼             ▼
  ✅ 成功撤离    💀 阵亡
  · 伤势保留     · 满血复活
  · 携带物品→仓库  · 携带物品全丢
  · 装备保留      · 装备全丢
  · 保险箱→仓库   · 保险箱→仓库
  · 获得经验      · 无经验
      │             │
      └──────┬──────┘
             │ ② 回到主界面
             ▼
        （循环重复）
```

### 2.2 循环驱动力

这个循环能跑起来，靠的是四个"推拉"力：

**推力1：伤势 → 治疗需求**
从地图出来不是满血的。必须用仓库里的药品手动治疗每个部位。如果药品不够 → 需要买 → 需要钱 → 需要再出击。

**推力2：装备 → 风险承担**
好装备提高生存率，但死了就没了。所以每次出击都是一场赌博——装备越好、入场费越高、赌注越大。

**拉力3：升级 → 解锁内容**
等级高了解锁新地图、商人新商品、新任务。这个"下一个解锁"的目标感让玩家持续玩。

**拉力4：保险箱 → 降低损失焦虑**
找到好东西可以塞保险箱，死了也不丢。但保险箱容量有限（初始2格，最高6格），迫使玩家取舍"这把要塞什么"。

### 2.3 一个典型玩家的一局（文字版）

> 小明在主界面用绷带把手臂治好，吃了压缩饼干补充体力。花12000买了突击步枪，3000买了轻甲，2000买了头盔。选了工厂（入场费3000）。
>
> 进图后往中心的高价值区走。路上踩了一个空物资格——浪费时间。又走一步碰到敌人，选择瞄准胸部攻击（命中率高），两枪打死，捡到金戒指和9mm弹药。
>
> 到了高价值区搜出显卡(¥18000)，包快满了。吃了MRE口粮补充体力。继续走发现上锁房间，钥匙没带，赌了一把破门——失败了，手臂撞伤15HP。
>
> 最后三回合找到撤离点，成功撤离。伤势保留（残血手臂），包里价值24000的物资入库。回到主界面用急救包治疗，卖了不用的弹药。
>
> 净赚：24000 - 3000(入场费) - 500(治疗开销) = 20500。

## 3. 系统设计

### 3.1 身体部位伤害系统

这是整个游戏最核心的系统，直接决定了战斗体验。

**部位列表：**

| 部位 | 满血 | 被命中权重 | 死亡条件 |
|------|------|-----------|---------|
| 头部 | 35 | 8%（最低） | ≤0 直接死亡 |
| 胸部 | 85 | 30%（最高） | ≤0 直接死亡 |
| 腹部 | 70 | 22% | 不致死 |
| 左臂 | 60 | 12% | 不致死 |
| 右臂 | 60 | 12% | 不致死 |
| 左腿 | 65 | 8% | 不致死 |
| 右腿 | 65 | 8% | 不致死 |

**死亡条件（满足任一即死）：**
- 头部归零
- 胸部归零
- 3个以上部位归零（失血过多）

**命中率公式：**

```
玩家命中率 = 武器精度 - 敌人闪避 + 距离加成/2
            └─ 范围限制在 [5%, 95%]

然后乘以部位系数：
  头部 ×0.45（最难命中）
  胸部 ×1.0
  腹部 ×0.85
  手臂 ×0.7
  腿部 ×0.65
```

**伤害计算：**

```
基础伤害 = 武器伤害 × [0.7, 1.3] 随机浮动

然后乘以部位倍率：
  头部 ×2.5（爆头）
  四肢 ×0.65（非致命部位）
  胸/腹 ×1.0

最终伤害 = 基础伤害 × (1 - 护甲值/100)
```

**敌人命中你的逻辑完全对称**——敌人也有精度/闪避/部位系数。这很重要，因为玩家和敌人共用一个伤害系统，玩家的"打头难"跟敌人的"打你头难"是对称的。

**断裂与修复：**
- 部位归零 = 断裂
- 普通药品无法治疗断裂部位（只能治疗1以上的部位）
- 断裂部位必须用手术包(`canFixBlacked=true`)修复
- 手术包修复后恢复到满血

**设计思想：**
- 头胸是"高风险目标"——想速杀就打头，但命中率低；想稳定就打胸
- 四肢是"保底选择"——命中率高但伤害低，适合收尾残血敌人
- 多部位断裂致死规则制造了"分散伤害"vs"集火一个部位"的策略选择

### 3.2 武器系统

**武器列表：**

| 武器 | 伤害 | 精度 | 弹药 | 价格 | 稀有度 |
|------|------|------|------|------|--------|
| 小刀 | 18 | 95% | 无 | 200 | 普通 |
| 手枪 | 35 | 75% | 9mm | 800 | 普通 |
| 冲锋枪 | 22 | 55% | 9mm | 3000 | 精良 |
| 霰弹枪 | 90 | 40% | 12g | 6000 | 精良 |
| 突击步枪 | 48 | 80% | 5.56 | 12000 | 稀有 |
| 精准步枪 | 75 | 90% | 7.62 | 25000 | 史诗 |
| 狙击步枪 | 150 | 95% | .338 | 50000 | 传说 |

**弹药：**

| 弹药 | 每盒数量 | 单价 | 适配武器 |
|------|---------|------|---------|
| 9mm | 30发 | 300 | 手枪/冲锋枪 |
| 5.56 | 30发 | 900 | 突击步枪 |
| 7.62 | 20发 | 1500 | 精准步枪 |
| 12g | 8发 | 500 | 霰弹枪 |
| .338 | 5发 | 5000 | 狙击步枪 |

**设计思考过程：**

武器定价不是按DPS公式算的，是一轮一轮手动调的。

狙击步枪50000：150×2.5爆头=375伤害，高级Boss也只有360HP左右。一枪一个Boss的能力值这个价。

霰弹枪6000是个尴尬的定位——伤害高(90)但精度低(40%)。实际测试中发现它在近距离（也就是每次战斗都是近距离，因为我们没有距离选择）表现极不稳定：打中了爽，打不中就白给。所以定价低于突击步枪。

冲锋枪精度55%是所有枪里最低的，但它便宜。新手过渡用。

弹药经济是关键约束。.338马格南一发1000块（5000/5发），狙击步枪每次攻击都可能浪费一发——你的命中率95%但敌人闪避后实际可能只有70%左右。所以用狙是豪赌。

**当前版本的不足：**
- 没有弹匣概念——弹药是直接消耗的，不够硬核。如果重来会加弹匣系统（弹匣容量×弹药消耗的逻辑）。
- 没有武器耐久——装备不会磨损，缺少"装备折旧"带来的经济压力。
- 没有配件系统（消音器/瞄具/握把）——这个做起来太复杂，HTML没法承载。

### 3.3 护甲系统

| 护甲 | 防护值 | 覆盖部位 | 耐久 | 价格 | 稀有度 |
|------|--------|---------|------|------|--------|
| 轻型防弹衣 | 20% | 胸/腹 | 60 | 3000 | 普通 |
| 中型防弹衣 | 40% | 胸/腹 | 80 | 12000 | 精良 |
| 重型防弹衣 | 65% | 胸/腹/双臂 | 120 | 40000 | 史诗 |
| 轻盔 | 15% | 头 | 40 | 2000 | 普通 |
| 战术头盔 | 35% | 头 | 60 | 10000 | 精良 |
| 重型头盔 | 55% | 头 | 90 | 35000 | 史诗 |

**减伤公式：** `实际伤害 = 基础伤害 × (1 - max(该部位所有护甲防护值) / 100)`

护甲取该部位最大值（不叠加），所以穿重甲+戴头盔就是65%胸腹保护+55%头部保护。

**设计思路：** 轻甲便宜但只护胸腹，重甲护四肢但贵。这个"花多少钱买多少保护"的梯度让装备选择有了意义。但护甲耐久目前没有实装——这是v1的遗憾。

### 3.4 医疗系统

| 药品 | 治疗量 | 使用次数 | 特殊能力 | 价格 | 稀有度 |
|------|--------|---------|---------|------|--------|
| 绷带 | 25 | 3次 | 无 | 300 | 普通 |
| AI-2急救包 | 50 | 2次 | 无 | 1200 | 精良 |
| Salewa医疗包 | 100 | 1次 | 无 | 6000 | 稀有 |
| 手术包 | 80 | 1次 | 修复断裂 | 15000 | 史诗 |

**局内vs局外治疗：**
- 局内：携带的医疗品直接使用，治疗指定部位
- 局外：从仓库选择医疗品，治疗指定部位。主界面每个部位旁边有[治疗]按钮

**批量治疗（局外）：** 主界面有"批量治疗"按钮，自动从便宜药品开始使用，治疗所有非断裂部位。

**为什么局内局外分开：**
这是我做了两版才确定的。第一版局内外共用同一套HP，结果玩家从地图出来就已经被藏身处的"战后回复"治好了，完全不需要手动用药品——医疗品成了废物。

第二版改成：撤离后伤势完全保留，藏身处医疗站的"战后回复"只是小幅辅助（20-100HP分散到全身），主力治疗还是要靠手动用药品。

这个改动的设计意图：**让"治伤"成为一个有意识的操作**，而不是自动发生的背景机制。玩家需要看着自己的残血身体部位一个个点治疗，这个过程本身就是"上次出击代价"的提醒。

### 3.5 背包/携带系统

| 背包 | 格数 | 价格 | 稀有度 |
|------|------|------|--------|
| 挎包（默认） | 4 | — | — |
| 拾荒者背包 | 8 | 3000 | 精良 |
| 战术背包 | 14 | 12000 | 稀有 |
| 突击背包 | 20 | 35000 | 史诗 |

胸挂（额外格子，与背包叠加）：

| 胸挂 | 格数 | 价格 |
|------|------|------|
| 轻型胸挂 | 2 | 800 |
| 战术胸挂 | 4 | 4000 |
| 突击胸挂 | 6 | 15000 |

总容量 = 背包格数 + 胸挂格数（无装备时默认4格挎包）。

**容量限制的设计价值：**
- 制造"捡什么留什么"的取舍——每次搜刮都是一次背包整理
- 驱动装备升级——"背包太小了得买个大的"
- 和保险箱容量形成双重约束——保险箱存贵重品，背包存一般物品

### 3.6 保险箱系统

**基本规则：**
- 保险箱内的物品阵亡不丢失
- 容量有限（初始2格，任务升级到3→4→6格）
- 局内找到的物品可以选择"存入保险箱"

**升级路径：**

| 等级 | 格数 | 解锁任务 |
|------|------|---------|
| Lv.1 | 2格 | 初始 |
| Lv.2 | 3格 | 完成3次成功撤离 |
| Lv.3 | 4格 | 在实验室成功撤离1次 |
| Lv.4 | 6格 | 击杀1个Boss |

**设计意图：**
保险箱解决的是"找到好东西反而更紧张"的问题。没有保险箱时，搜到传说级物品（LEDX ¥80000、黄金头骨 ¥100000）后玩家只想直奔撤离点，中间的所有探索和战斗都变成了"威胁"而非"机会"。

2格保险箱只能存1个传说+1个史诗——玩家还是需要做选择。这个选择本身就是游戏体验的一部分。

高级保险箱需要完成有难度的任务（实验室撤离、Boss击杀），这给"升级保险箱"赋予了成就感和进度感。

### 3.7 地图系统

**四张地图设计：**

| 地图 | 尺寸 | 回合 | 敌人 | 入场费 | 等级 | 掉落倾向 |
|------|------|------|------|--------|------|---------|
| 城郊住宅区 | 6×6 | 12 | 3-6 | 0 | Lv1 | 基础物资，少量稀有 |
| 废弃工厂 | 8×8 | 18 | 5-10 | 3000 | Lv3 | 钥匙/材料/零件丰富 |
| 中央商场 | 10×10 | 24 | 8-14 | 8000 | Lv6 | 奢侈品/电子产品 |
| 地下实验室 | 12×12 | 30 | 10-20 | 25000 | Lv10 | 顶级装备/传说级 |

**地图生成规则（每次随机）：**
1. 出生点：左上角随机格
2. 撤离点：右下角区域，3个（开局可见）
3. 高价值区：地图正中心3×3区域（开局可见 💎图标）——掉落率+15%
4. 危险区域：踩上受到8-20伤害，2-8个随机分布
5. 上锁房间：1-5个随机分布，需钥匙或强行破门
6. Boss：每个地图必有一个，开局可见 👹图标，在离出生点较远的位置
7. 普通敌人：3-20个随机分布（取决于地图）
8. 物资：lootBudget约30-120个物品，分布在全图

**回合限制的设计：**

每张图的回合数 = 从出生点到对角撤离点的最远步数 + 缓冲。

- 6×6：对角线10步 + 2缓冲 = 12回合
- 8×8：对角线14步 + 4缓冲 = 18回合
- 10×10：对角线18步 + 6缓冲 = 24回合
- 12×12：对角线22步 + 8缓冲 = 30回合

回合归零后触发MIA警告（下次没到撤离点就死）。这个机制让"先搜后撤"的时间分配成为一个策略选择——搜太多可能来不及撤。

**高价值区为什么开局可见：**
文字游戏没有FPS的"视觉引导"——你不能看到远处有灯光或听到声音来判断哪里有好东西。如果不显示高价值区，玩家只能盲目乱走，体验极差。所以开局就显示💎——"好地方在中心，去不去你自己决定"。

### 3.8 战斗系统

**战斗是遭遇式的**——踩到敌人格子即触发，弹出战斗界面。

**战斗面板布局：**
```
┌────────────────────────────┐
│       👹 敌人名称          │
│       ████████░░ HP条      │
│                            │
│  [🧠头部 35/35] [❤️胸部... │
│  [💪左臂 60/60] [💪右臂... │
│  [🫁腹部 70/70] [🦵左腿... │
│  [🦵右腿 65/65]            │
│                            │
│  点击部位瞄准攻击           │
│  [🏃逃跑]                  │
│                            │
│  ── 战斗日志 ──            │
│  👊 你击中胸部 造成48伤害   │
│  💢 敌人击中左臂 造成12伤害  │
└────────────────────────────┘
```

**战斗流程：**
1. 玩家选择一个身体部位攻击
2. 计算命中（武器精度 - 敌人闪避）× 部位系数
3. 命中 → 计算伤害 × 部位倍率 → 扣敌人HP
4. 敌人死亡 → 自动获得战利品 → 关闭战斗面板 → 弹出拾取UI（如果有物品）
5. 敌人没死 → 敌人反击（同样的命中/伤害逻辑）→ 回到步骤1

**逃跑机制：**
- 基础成功率50%
- Boss降低20%（即30%成功率）
- 失败后敌人立即攻击，然后继续战斗

**敌人数据：**

| 难度 | 普通敌人HP | 伤害 | 精度 | Boss HP | Boss伤害 | Boss精度 |
|------|-----------|------|------|---------|---------|---------|
| 1(城郊) | 30-55 | 8-15 | 25-38% | 120-160 | 20-35 | 50-60% |
| 2(工厂) | 55-90 | 15-22 | 33-48% | 180-240 | 35-55 | 60-70% |
| 3(商场) | 80-120 | 22-29 | 41-58% | 240-320 | 55-75 | 70-80% |
| 4(实验室) | 105-155 | 29-36 | 49-68% | 320-400 | 75-95 | 80-90% |

（实际数值有少量随机浮动）

### 3.9 经济系统

**完整的经济流：**

```
       搜刮物资
          │
    ┌─────┴─────┐
    ▼           ▼
卖给商人    卖到跳蚤
(60%基础价)  (80%~120%基础价)
    │           │
    └─────┬─────┘
          ▼
        金币
          │
    ┌─────┼─────┬─────┐
    ▼     ▼     ▼     ▼
  买装备  付入场费 升级商人 升级藏身处
 (130%价)
```

**为什么商人回收价只有60%而卖价130%？**
制造70%的差价 = 制造经济压力。如果回收价=卖价，搜一局能直接买回来，没有任何经济决策。差价逼着玩家持续搜刮，而且"丢装备"的代价变大了——你花13000买的突击步枪，丢了等于丢了7800（60%回收价）的实际价值。

**看广告：**
- 基础奖励 500 + 等级×200 + 随机0-500
- 5分钟冷却
- 10%概率额外获得物品（绷带、弹药等）
- 这就是个游戏内福利，不是真实的广告（HTML做不到）

**体力/水分的经济影响：**
- 体力和水分在局外不会自动回复
- 必须吃食物（从仓库或购买）
- 压缩饼干¥100/15体力 → 成本约¥7/体力
- 大图22步消耗44体力 = 约¥300的食物成本
- 这给"出击"增加了一个稳定的运营成本

### 3.10 商人系统

**四个商人的定位：**

| 商人 | 定位 | 特色商品 |
|------|------|---------|
| 和平使者 | 进口军需 | 手枪/冲锋枪/步枪/轻甲/9mm/5.56 |
| 帕拉波尔 | 俄式重火力 | DMR/狙击/重甲/7.62/.338 |
| 治疗师 | 医疗供应 | 绷带→手术包/食物/医疗箱 |
| 滑雪者 | 黑市 | 全武器随机/重甲/贵重品/最大背包 |

**升级条件：**
- 每个商人4级
- 升级需要：①在该商人处累计消费达标 + ②支付升级费用
- 每级解锁新商品（例如和平使者Lv2解锁DMR和M855弹药）

**升级消费门槛：**

| 等级 | 和平使者 | 帕拉波尔 | 治疗师 | 滑雪者 |
|------|---------|---------|--------|--------|
| 1→2 | 消费2万+5千 | 3万+8千 | 1万+3千 | 5万+1万 |
| 2→3 | 10万+3万 | 15万+4万 | 5万+1.5万 | 20万+5万 |
| 3→4 | 50万+10万 | 80万+15万 | 20万+5万 | 100万+20万 |

**为什么滑雪者的门槛最高：** 他是"后期内容"——卖传说装备和最大背包。门槛保证玩家在游戏前期不能直接跳到最好装备。

### 3.11 藏身处系统

| 设施 | 最大等级 | 满级效果 | 主要材料 |
|------|---------|---------|---------|
| 发电机 | 3 | 电力3 | 零件/电线/马达 |
| 工作台 | 3 | 修理加成50% | 零件/马达/电路板 |
| 医疗站 | 3 | 战后自动回复100HP | 零件/电路板/马达 |
| 情报中心 | 3 | 入场费-50%+掉落+25% | 电路板/传感器/马达 |
| 储物箱 | 3 | 仓库+50格 | 零件/电路板/传感器 |

**材料用途表：**

| 材料 | 用途 | 稀有度 | 来源 |
|------|------|--------|------|
| 零件 | 所有设施基础材料 | 普通 | 地图掉落(高概率)/敌人掉落 |
| 电线 | 低级设施 | 普通 | 地图掉落 |
| 电路板 | 中级设施 | 精良 | 地图掉落(低概率) |
| CPU风扇 | 中级设施 | 精良 | 地图掉落 |
| 马达 | 高级设施 | 稀有 | 地图掉落(低概率) |
| 传感器 | 顶级设施 | 史诗 | 地图掉落(很低概率) |

**设计意图：**
藏身处解决了"捡的垃圾有啥用"的问题。如果所有搜刮品只能卖钱，那"零件¥150"看起来毫无吸引力。但如果你需要30个零件升发电机，那每个零件都变成了"进度条的一部分"——赋予了垃圾价值感。

### 3.12 任务系统

| 任务 | 条件 | 奖励 | 等级要求 |
|------|------|------|---------|
| 初次出击 | 完成1次成功撤离 | ¥3000 | Lv1 |
| 清道夫 | 累计击杀5个敌人 | ¥5000 | Lv1 |
| 发财了 | 单局搜刮超过¥10000 | ¥8000 | Lv3 |
| 保险箱升级I | 完成3次成功撤离 | 保险箱升2级(3格) | Lv5 |
| 战争机器 | 累计击杀20个敌人 | ¥20000 | Lv7 |
| 保险箱升级II | 实验室成功撤离1次 | 保险箱升3级(4格) | Lv10 |
| 屠龙者 | 击杀1个Boss | 保险箱升4级(6格) | Lv12 |
| 亿万富翁 | 累计搜刮¥500000 | ¥100000 | Lv15 |

**任务系统的设计原则：**
- 前两个任务（初次出击、清道夫）门槛极低——让玩家快速获得"完成任务"的成就感
- 保险箱升级任务形成一条进度线——"我要升级保险箱"成为中期核心目标
- Boss击杀是"能力检验"——需要好装备+好药品+好运气

### 3.13 破门机制

上锁房间需要钥匙才能打开。没有钥匙时可以选择"强行破门"。

| 方式 | 成功率 | 成本 |
|------|--------|------|
| 用钥匙 | 100% | 消耗钥匙 |
| 强行破门 | 30% | 失败 → 弹回原位 + 手臂/胸部10-25伤害 |

**设计过程：**
第一版失败只是弹回——没有惩罚，纯正面赌博。这是设计失误。

第二版加了受伤惩罚。10-25伤害可以接受（手臂断了用手术包，胸伤了用急救包），但配合敌人战斗和地图危险区，这个额外伤害就有了连锁反应——破门失败的10点伤害可能让你下一场战斗恰恰卡在死亡线上。

关于30%的概率：
- 太高（>50%）：钥匙完全没价值，没人会带钥匙
- 太低（<15%）：没人会去赌，这个机制等于不存在
- 30%：概率够低所以大部分时候失败，但"万一成功了呢"的心理驱动玩家去赌

### 3.14 存档系统

- **自动存档：** 每次操作后自动 `localStorage.setItem()`
- **导出：** JSON文件下载，可以用记事本打开修改（某种意义上算作弊，但单机游戏无所谓）
- **导入：** 选择JSON文件读回
- **重置：** 确认后清空localStorage

存档不包括当前的突袭（`delete G.currentRaid`），防止在局内战死时强退再读档。

## 4. 数值设计

### 4.1 升级曲线

`xpForLevel(Lv) = floor(100 × 1.5^(Lv-1))`

| Lv | 升级需要 | 累计XP | 预计撤离次数 |
|----|---------|--------|------------|
| 1→2 | 100 | 100 | 1-2 |
| 3→4 | 225 | 475 | 5-6 |
| 5→6 | 506 | 1329 | 13-15 |
| 8→9 | 1708 | 5126 | 50-60 |
| 10→11 | 3844 | 11333 | 100-120 |
| 12→13 | 8649 | 22735 | 200-230 |
| 15→16 | 29194 | 87384 | ~800 |

**经验来源：** 成功撤离50 + 物品价值/100 + 击杀数×20。

以工厂为例（中等难度图）：成功撤离50 + 8000/100 + 3击杀×60 = 190XP。前10级约需60局，合理。

### 4.2 掉落概率

| 稀有度 | 稀有度值 | 基础概率 | 高价值区概率 | 情报中心满级 |
|--------|---------|---------|------------|------------|
| 普通 | 1× | ~40% | ~25% | ~28% |
| 精良 | 2× | ~30% | ~20% | ~22% |
| 稀有 | 4× | ~15% | ~25% | ~28% |
| 史诗 | 8× | ~3% | ~15% | ~20% |
| 传说 | 20× | ~0.5% | ~4% | ~8% |

掉落公式用了两级过滤：先按概率随机稀有度，再用池子权重过滤一次（防止稀有物品泛滥）。

### 4.3 武器DPS对比（回合制下的等效）

假设打胸部（命中率=武器精度，没有敌人闪避的理想情况）：

| 武器 | 期望每回合伤害 | 千伤害成本 |
|------|-------------|-----------|
| 小刀 | 18×0.95 = 17.1 | ¥11.7 |
| 手枪 | 35×0.75 = 26.3 | ¥30.4 |
| 冲锋枪 | 22×0.55 = 12.1 | ¥248（太低了，但射速5可能命中多次） |
| 突击步枪 | 48×0.80 = 38.4 | ¥313 |
| 霰弹枪 | 90×0.40 = 36.0 | ¥167 |
| 精准步枪 | 75×0.90 = 67.5 | ¥370 |
| 狙击步枪 | 150×0.95 = 142.5 | ¥351 |

冲锋枪期望伤害最低，但因为射速高（5 vs 步枪的3），在一个回合内可以命中更多次——这在目前的实现中体现为多次伤害判定，但因为命中率只有55%，实际体验很差。我认为这是个设计问题，可能需要提高冲锋枪的精度或者降低其他武器的精度。

## 5. UI/UX 设计

### 5.1 色彩系统

- 深紫色渐变背景（模仿塔科夫的深色调）
- 5级稀有度色：灰/绿/蓝/紫/金
- 粉色主色调用于按钮和强调
- HP条：绿(>60%)→橙(30-60%)→红(<30%)

### 5.2 设计原则

1. **所有操作不超过2次点击**——没有深层嵌套菜单
2. **重要信息始终可见**——地图上高价值区/撤离点/Boss开局就显示
3. **状态变化有颜色反馈**——HP条颜色变化、战利品闪烁、Boss脉冲动画
4. **不影响主界面的弹出面板**——战斗/拾取/治疗都是overlay，关掉就回到地图

### 5.3 移动端适配

v1只做了基础的responsive。600px以下缩小字体和间距。如果要正式上线，移动端需要大幅改版——目前的网格在手机上太小。

## 6. 技术实现说明

### 6.1 架构

- 单文件HTML（3个语言块：CSS + HTML + JS）
- 全局状态对象 `G` 包含所有游戏状态
- 渲染是"全量替换"模式——`renderAll()` 重绘整个界面
- 没有虚拟DOM、没有React、没有任何框架

### 6.2 为什么选这个技术栈

1. 我不需要学Vue/React来做一个回合制小游戏
2. 单文件方便展示——面试官双击就能打开
3. localStorage不需要后端——不需要解释"为什么没有服务器"

### 6.3 我学到了什么

- JS的变量提升和作用域——`var` vs `let` 的坑踩了好几次
- 事件处理中的闭包问题——onclick里引用循环变量时的经典bug
- CSS Grid做棋盘地图很方便
- 全量渲染的性能比我想象的好——800格的地图帧率完全没问题

## 7. 已知问题和改进方向

### 7.1 当前版本的问题

1. **战斗没有距离维度**——所有战斗都是"近战"，武器精度里的"距离"因素没用上
2. **没有武器耐久**——装备永久使用，缺少折旧压力
3. **弹药没有弹匣概念**——打一发扣一发，不是弹匣打空换弹
4. **移动端体验差**——12×12地图的格子太小
5. **没有音效**——文字游戏缺少反馈感
6. **没有新手引导**——玩家上来不知道要干嘛
7. **数值没有做过AB测试**——所有数值都是"感觉差不多"
8. **冲锋枪太弱**——低精度+低伤害，性价比极差

### 7.2 如果有下一版

1. **弹匣系统：** 弹匣有容量，打空需要换弹（消耗一个回合）
2. **武器耐久/卡壳：** 低耐久武器有概率卡壳，需要工作台修理
3. **距离系统：** 战斗开始时根据武器射程决定先手权
4. **异步PVP：** 其他玩家的尸体作为AI敌人出现在地图中
5. **赛季重置：** 每3个月清零经济，保留皮肤/成就
6. **移动端原生适配：** 触屏手势、更大的点击区域
